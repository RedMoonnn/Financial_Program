# 项目 To-Do List

## 1. 代码异步化改造
将现有的同步阻塞代码重构为异步非阻塞模式，以提升并发处理能力和系统响应速度。具体包括：
- **大模型访问 (LLM Access)**:
  - 将 `DeepseekAgent` 中的 HTTP 请求改为使用 `aiohttp` 或 `httpx` 的异步客户端。
  - 确保 API 调用不会阻塞事件循环。
- **爬虫函数调用**:
  - 重构 `crawler` 模块，使用异步 IO 处理网络请求。
  - 在 Celery 任务或后台任务中正确处理异步调用。
- **数据库操作**:
  - 全面检查并确保所有数据库交互使用 SQLAlchemy 的 `async` 扩展。

## 2. 压力测试 (Stress Testing)
对系统进行全面的压力测试，以评估在高并发情况下的稳定性和性能瓶颈。
- **测试场景**:
  - 多用户并发生成报告。
  - 高频次爬虫任务触发。
  - 大量并发聊天请求。
- **指标关注**:
  - 系统吞吐量 (TPS/QPS)。
  - 接口响应时间 (Latency)。
  - 资源占用 (CPU, Memory, I/O)。
- **工具**: 使用 Locust 或 JMeter。

## 3. Agent 优化
优化 AI Agent 的智能程度和执行效率。
- **Prompt 优化**: 迭代提示词，提高回答的准确性和结构化程度。
- **上下文管理**: 优化历史对话的存储和检索策略，减少 Token 消耗并保留关键信息。
- **工具调用**: 增强 Agent 调用外部工具（如实时查询、数据库检索）的能力。
- **流式输出**: 完善前端的流式响应体验，确保在各种网络环境下都能平滑显示。
